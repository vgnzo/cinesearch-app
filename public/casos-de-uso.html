Detalhamento dos Casos de Uso

Fluxo, camadas envolvidas e tecnologia

IDNomeFluxo PrincipalCamadas
UC01Cadastrar Filme Usuário (logado) preenche formulário → Frontend envia POST com token JWT → Backend valida token + regras → Elasticsearch cria documento com autor_id Front Back DB
UC02Buscar Filme Usuário digita termo → Frontend envia GET com ?q= → Backend monta query multi_match → Elasticsearch busca em 3 campos e retorna resultados Front Back DB
UC03Filtrar por Gênero Usuário clica no chip → Frontend preenche o campo de busca com o gênero → Mesmo fluxo do UC02 com o gênero como termo Front Back DB
UC04Ver Todos Usuário clica em "Ver Todos" → Frontend envia GET sem parâmetro → Backend usa match_all → Elasticsearch retorna todos os documentos Front Back DB
UC05Editar Filme Usuário clica em Editar → Formulário preenchido → Salva com token → Backend verifica se autor_id = usuário logado → Elasticsearch atualiza documento Front Back DB
UC06Deletar Filme Usuário clica em Deletar → Frontend envia DELETE com token → Backend verifica se autor_id = usuário logado → Elasticsearch remove permanentemente Front Back DB
UC07Registrar Usuário Usuário preenche nome, email e senha → Backend valida campos → bcrypt criptografa a senha → Elasticsearch salva no índice "usuarios" Front Back DB
UC08Fazer Login Usuário informa email e senha → Backend busca usuário no ES → bcrypt compara senhas → JWT gera token → Frontend salva token no localStorage Front Back DB
UC09Fazer Logout Usuário clica em Sair → Frontend remove token e dados do localStorage → UI atualizada para estado deslogado Front
